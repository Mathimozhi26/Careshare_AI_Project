ISSUE #1: MongoDB Connection Not Established
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Severity: CRITICAL
Component: Backend Server
Category: Infrastructure
Status: BLOCKING ALL TESTS

Description:
  The backend server cannot establish connection to MongoDB at the configured URI:
  mongodb://127.0.0.1:27017/caresure
  
  This prevents:
  - Server from starting up
  - All database operations
  - All API tests that depend on data persistence
  - Authentication tests that store user data

Root Causes:
  1. MongoDB is not running locally on port 27017
  2. MongoDB service is not started on the system
  3. Incorrect MONGO_URI in .env file
  4. Firewall/network blocking access to MongoDB

Quick Fix:
  Step 1: Install MongoDB if not present
    Download from: https://www.mongodb.com/try/download/community
    
  Step 2: Start MongoDB service
    On Windows: 
      - Open Services app (services.msc)
      - Find "MongoDB Server" service and Start it
      - Or run: mongod.exe from command line
      
  Step 3: Verify connection
    - Open MongoDB Compass
    - Try connecting to mongodb://127.0.0.1:27017
    - Should show connected status

  Alternative: Use MongoDB Atlas (Cloud)
    Step 1: Create account at https://www.mongodb.com/cloud/atlas
    Step 2: Create a free cluster
    Step 3: Update MONGO_URI in .env:
      MONGO_URI=mongodb+srv://user:password@cluster.mongodb.net/caresure
    Step 4: Restart backend

ISSUE #2: Port 5000 Already in Use
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
Severity: CRITICAL
Component: Backend Server
Category: Infrastructure
Status: BLOCKING BACKEND START

Description:
  Another process is already using port 5000, preventing the Express server
  from binding to it.

Root Causes:
  1. Previous backend instance still running
  2. Another application using port 5000
  3. Port not released after previous shutdown

Quick Fix:
  Option 1: Kill process using port 5000
    On Windows PowerShell:
      # Find process on port 5000
      Get-NetTCPConnection -LocalPort 5000 | Select-Object OwningProcess
      # Kill the process (replace PID with actual process ID)
      Stop-Process -Id <PID> -Force
      
  Option 2: Use different port
    Edit .env and change:
      PORT=5000  →  PORT=5001
    Then restart backend

  Option 3: Restart computer
    Cleanly resets all port bindings
